Responsive Design - Breakpoints

Breakpoints (Puntos de quiebre)

Prefijo Tamaño Mínimo Dispositivo Media Query CSS
sm 640px Móvil horizontal / Tablet pequeña @media (min-width: 640px)
md 768px Tablet @media (min-width: 768px)
lg 1024px Laptop @media (min-width: 1024px)
xl 1280px Desktop @media (min-width: 1280px)
2xl 1536px Desktop grande / 4K @media (min-width: 1536px)

⚠️ Mobile First: Tailwind usa el enfoque "mobile first". Las clases sin prefijo aplican a móviles. Los prefijos aplican estilos "desde ese tamaño hacia arriba".

Ejemplos Prácticos

1. Tamaño de texto responsive

Este texto crece según el tamaño de pantalla

class="text-sm sm:text-base md:text-lg lg:text-xl xl:text-2xl"

Móvil: text-sm → SM: text-base → MD: text-lg → LG: text-xl → XL: text-2xl

2. Grid responsive (columnas adaptativas)

Caja 1
Caja 2
Caja 3
Caja 4
class="grid grid-cols-1 sm:grid-cols-2 md:grid-cols-3 lg:grid-cols-4"

Móvil: 1 columna → SM: 2 columnas → MD: 3 columnas → LG: 4 columnas

3. Ocultar/Mostrar elementos según pantalla

Solo visible en móvil y SM (oculto desde MD)
Solo visible desde MD en adelante
Solo visible desde LG en adelante
md:hidden → Oculta desde MD
hidden md:block → Oculto en móvil, visible desde MD
hidden lg:block → Oculto hasta LG

4. Espaciado responsive

Mi padding crece con la pantalla
class="p-2 sm:p-4 md:p-6 lg:p-8 xl:p-10"

5. Dirección de Flexbox responsive

Elemento 1
Elemento 2
Elemento 3
class="flex flex-col md:flex-row"

Móvil: columna vertical → MD: fila horizontal

6. Indicador de breakpoint actual (para testing)

XS (menos de 640px) SM (640px+) MD (768px+) LG (1024px+) XL (1280px+) 2XL (1536px+)

RESUMEN RÁPIDO

  • ✅ Mobile First: Diseña primero para móvil, luego agrega breakpoints
  • ✅ Sintaxis: prefijo:clase → Ejemplo: md:text-lg
  • ✅ Sin prefijo = móvil → Aplica a todos los tamaños
  • ✅ Con prefijo = desde ese tamaño → md:block = visible desde 768px
  • ✅ Se pueden combinar: hidden md:block lg:flex xl:grid
  • ✅ Prueba responsive: DevTools → Toggle device toolbar (Ctrl+Shift+M)
